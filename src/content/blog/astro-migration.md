最近，我决定对我的个人博客进行一次彻底的“大手术”——将框架从 Hexo 切换到了 Astro。

### 为什么要迁移？

Hexo 是一个非常成熟的静态网站生成器，但随着前端技术的发展，我逐渐发现了一些局限性：
1. **性能瓶颈：** 随着文章增多，生成速度变慢。
2. **定制化困难：** 想要修改 UI 样式往往需要深入研究复杂的模板引擎。
3. **现代技术栈支持：** 我希望能更方便地使用 Tailwind CSS 和现代化的组件库。

### 为什么选择 Astro？

Astro 是一个专为内容驱动型网站设计的现代框架。它的核心优势在于：
- **极速性能：** 默认发送零 JS 到客户端，加载速度极快。
- **灵活的组件系统：** 支持 React, Vue, Svelte 等多种框架，也可以直接使用原生 HTML/CSS。
- **优秀的开发者体验：** 内置了对 Markdown, MDX 和 Tailwind 的完美支持。

### 迁移过程

1. **内容备份：** 将原有的 HTML 和静态资源进行备份。
2. **初始化 Astro：** 使用 `npm create astro@latest` 快速搭建项目。
3. **内容迁移：** 将 HTML 内容转换为 Markdown 格式，并统一 Frontmatter 格式。
4. **UI 重塑：** 使用 Tailwind CSS 设计了一套简约现代的玻璃拟态（Glassmorphism）风格。

### 总结

这次迁移不仅仅是工具的更换，更是一次对个人博客意义的重新思考。新的框架让我能更专注于内容创作，同时也提供了更好的阅读体验。
